# 獨熱編碼(One-Hot Encoding)

------

## 一句話先懂

**one-hot coding** 就是：

> 「用一排只有一個 `1`、其他全部是 `0` 的數字，來代表一個類別。」

------

## 用超生活化的例子 🌈

假設你只想表示三種水果：

- 🍎 蘋果
- 🍌 香蕉
- 🍊 橘子

我們先幫它們「排位置」：

| 水果 | 位置 |
| ---- | ---- |
| 蘋果 | 第 1 |
| 香蕉 | 第 2 |
| 橘子 | 第 3 |

------

## one-hot 是怎麼表示的？

規則只有一個：

> **屬於哪一類，就在那個位置放 `1`，其他全部放 `0`**

| 水果   | one-hot 表示 |
| ------ | ------------ |
| 🍎 蘋果 | `[1, 0, 0]`  |
| 🍌 香蕉 | `[0, 1, 0]`  |
| 🍊 橘子 | `[0, 0, 1]`  |

🔹 你會發現：

- **永遠只有一個 `1`**
- 其他都是 `0`
- 所以叫 **one-hot（一個是熱的）**

------

## 為什麼不能直接用數字？

你可能會想：

> 蘋果=1，香蕉=2，橘子=3 不就好了？

⚠️ 問題在這裡：

對電腦來說：

- `3` 看起來比 `1` 大
- 但 **橘子沒有比蘋果「大 3 倍」**
- 類別之間 **沒有大小、距離關係**

one-hot 的好處是：

- 每個類別 **完全平等**
- 不會讓模型誤以為「香蕉介於蘋果與橘子之間」

------

## 跟「文字」有什麼關係？ 📚

假設一句話：

> ```
> 我 愛 你
> ```

先給每個字一個編號：

| 字   | 編號 |
| ---- | ---- |
| 我   | 1    |
| 愛   | 2    |
| 你   | 3    |

用 one-hot 表示：

- 我 → `[1, 0, 0]`
- 愛 → `[0, 1, 0]`
- 你 → `[0, 0, 1]`

👉 這就是 **最原始的文字表示法**

------

## 但它有一個超大的缺點 😵

如果：

- 字典有 **50,000 個字**
- one-hot 向量就有 **50,000 個數**
- 而且 **只有 1 個是 1，其餘全是 0**

➡️ **超浪費空間、超慢**

這正是為什麼會出現：

> **Embedding layer**

------

## 超重要連結（幫你鋪路）

現在你可以這樣理解整件事：

- one-hot：
  👉「只是告訴電腦：這是哪一個字」
- embedding：
  👉「把這個字變成一個有意義、會學習的向量」

Embedding = one-hot 的 **升級版、省電版、聰明版**
